FTP Network Lab

Red virtual con VirtualBox, Windows Server 2019, IIS y FileZilla · Networking

Módulo Networking & Infrastructure
Institución University of Wales Trinity Saint David / CESTE
Estado Completado
Stack
VirtualBox Windows Server 2019 IIS FileZilla Server FTP/FTPS Active Directory

El problema

El objetivo era diseñar e implementar una red local virtualizada con un servidor FTP funcional, simulando un entorno empresarial real. El sistema debía permitir la transferencia segura de archivos entre máquinas virtuales con control de acceso por usuarios y permisos por directorio.

La restricción principal era implementarlo exclusivamente con herramientas de Windows Server, sin software adicional de terceros salvo FileZilla como cliente FTP para las pruebas.

Proceso de desarrollo

01

Diseño de la topología de red

Configuración de VirtualBox con dos VMs: servidor (Windows Server 2019) y cliente (Windows 10). Red interna con adaptador host-only para aislamiento del entorno y asignación de IPs estáticas.

02

Instalación y configuración de Windows Server 2019

Instalación del sistema operativo servidor, configuración de red, habilitación de roles necesarios mediante Server Manager: Web Server (IIS) y FTP Server.

03

Configuración del servicio FTP en IIS

Creación del sitio FTP en IIS Manager, enlace al puerto 21, configuración de directorio raíz, habilitación de SSL/TLS para FTPS y configuración del firewall de Windows para permitir el tráfico FTP.

04

Gestión de usuarios y permisos

Creación de usuarios locales en Windows Server, asignación de permisos de lectura/escritura por directorio mediante IIS Authorization Rules. Configuración de directorios virtuales por usuario con aislamiento de sesión.

05

Pruebas con FileZilla

Verificación de conexión FTP y FTPS desde la VM cliente usando FileZilla. Pruebas de subida, descarga, creación de directorios y verificación de que los permisos por usuario funcionan correctamente.

Lo que aprendí

Configurar un servidor FTP desde cero en Windows Server me dio una comprensión real de cómo funcionan los protocolos de transferencia de archivos a nivel de infraestructura — algo que los servicios cloud abstraen completamente. Entender la diferencia entre FTP, FTPS y SFTP, y por qué el FTP sin cifrado es inaceptable en producción, fue un aprendizaje clave de seguridad.

También fue mi primera experiencia seria con virtualización de red, un fundamento esencial para cualquier rol de IT o DevOps.
